# 在线商城系统 - 开发计划书

## 一、项目概述

### 1.1 项目背景
使用Python语言开发一个完整的在线商城系统，作为课程设计/毕业设计项目。

### 1.2 技术栈
| 技术 | 选型 | 说明 |
|------|------|------|
| 语言 | Python 3.10+ | 主开发语言 |
| Web框架 | Django 4.2 | 全功能框架，自带ORM/Admin/用户认证 |
| 前端 | Django Template + Bootstrap 5 | 前后端一体，快速开发 |
| 数据库 | PostgreSQL | 关系型数据库 |
| 图片处理 | Pillow | 商品图片处理 |
| 表单优化 | django-crispy-forms | 表单渲染美化 |

---

## 二、功能模块

### 2.1 用户模块
- 用户注册（用户名、邮箱、密码）
- 用户登录/登出
- 个人中心（修改信息、修改密码、收货地址管理）

### 2.2 商品模块
- 商品分类（支持多级分类）
- 商品列表（分页、分类筛选、关键词搜索）
- 商品详情页（图片轮播、描述、库存、价格）
- 新品上架 / 热门推荐

### 2.3 购物车模块
- 加入购物车
- 修改商品数量
- 删除购物车商品
- 全选/取消全选
- 未登录用户使用Session存储，登录后自动合并

### 2.4 订单模块
- 从购物车结算生成订单
- 填写收货地址
- 订单状态管理：待支付 → 已支付 → 已发货 → 已完成 / 已取消
- 订单列表（分页、状态筛选）
- 订单详情

### 2.5 支付模块
- 模拟支付（显示支付页面，确认后更新状态）
- 支付记录存储

### 2.6 评价模块
- 仅已购买用户可评价
- 评分（1-5星）+ 文字评价
- 商品详情页展示评价列表和平均评分

### 2.7 后台管理
- 商品管理（增删改查、上下架）
- 订单管理（查看、修改状态）
- 用户管理
- 评价管理
- 数据统计仪表盘（商品数、订单数、用户数、销售额）

## 四、数据库设计

### 4.1 核心模型

#### 用户相关
```python
# User (Django自带)
# - username, email, password

# Profile (OneToOne -> User)
# - avatar, phone, birth_date

# Address (ForeignKey -> User)
# - name, phone, province, city, district, detail, is_default
```

#### 商品相关
```python
# Category
# - name, parent(自关联), icon, sort_order

# Goods
# - name, description, price, stock, category(FK)
# - image, is_active, created_at, updated_at
```

#### 购物车
```python
# Cart (ForeignKey -> User)
# - created_at, updated_at

# CartItem (ForeignKey -> Cart, ForeignKey -> Goods)
# - quantity
```

#### 订单
```python
# Order
# - order_no(唯一订单号), user(FK)
# - total_amount, status(待支付/已支付/已发货/已完成/已取消)
# - receiver_name, receiver_phone, receiver_address
# - created_at, paid_at, shipped_at

# OrderItem (ForeignKey -> Order, ForeignKey -> Goods)
# - goods_name, goods_price, goods_image (快照)
# - quantity
```

#### 支付
```python
# Payment (OneToOne -> Order)
# - payment_no, amount, method, status, paid_at
```

#### 评价
```python
# Review
# - user(FK), goods(FK), order(FK)
# - rating(1-5), content, created_at
# - unique_together: (user, goods) — 一个用户对一个商品只能评价一次
```

### 4.2 ER关系图
```
User ──1:1──> Profile
User ──1:N──> Address
User ──1:N──> Cart ──1:N──> CartItem ──N:1──> Goods
User ──1:N──> Order ──1:N──> OrderItem ──N:1──> Goods
Order ──1:1──> Payment
User ──1:N──> Review ──N:1──> Goods
Category ──1:N──> Goods
Category ──1:N──> Category (自关联，支持多级分类)
```

## 五、开发步骤

### 步骤1: 项目初始化
- 创建Django项目和app目录结构
- 配置PostgreSQL数据库连接
- 配置静态文件、媒体文件路径
- 安装依赖：`pip install -r requirements.txt`

### 步骤2: 用户模块
- 扩展User模型（Profile、Address）
- 实现注册、登录、登出视图
- 编写表单验证
- 创建用户相关模板页面

### 步骤3: 商品模块
- 创建Category和Goods模型
- 编写商品列表视图（分页、筛选、搜索）
- 编写商品详情视图
- 配置Admin后台管理

### 步骤4: 购物车模块
- 创建Cart和CartItem模型
- 实现加购、修改数量、删除操作
- 添加全局Context Processor（导航栏显示购物车数量）
- Session购物车与登录用户购物车合并逻辑

### 步骤5: 订单模块
- 创建Order和OrderItem模型
- 实现下单流程（购物车→结算→确认→生成订单）
- 库存校验与扣减
- 订单列表和详情页面
- 订单状态流转

### 步骤6: 支付模块
- 创建Payment模型
- 实现模拟支付页面和流程
- 支付成功后更新订单状态

### 步骤7: 评价模块
- 创建Review模型
- 实现发表评价功能
- 商品详情页展示评价

### 步骤8: 后台管理 + 数据统计
- 定制Django Admin（各模型的list_display、search_fields、filters）
- 订单状态流转管理
- 首页数据统计

### 步骤9: 前端美化
- 统一页面风格（Bootstrap 5）
- 首页设计（轮播、推荐商品）
- 响应式布局适配
- 表单验证提示、操作反馈

## 七、验证方案

1. **环境启动**: `pip install -r requirements.txt` → `python manage.py migrate` → `python manage.py createsuperuser` → `python manage.py runserver`
2. **完整流程测试**:
   - 注册新用户 → 登录 → 浏览商品 → 加入购物车 → 去结算 → 填写地址 → 下单 → 模拟支付 → 查看订单 → 发表评价
3. **后台管理测试**: 登录Admin → 管理商品/订单/用户/评价
4. **数据库验证**: 检查PostgreSQL中表结构和数据一致性
5. **边界测试**: 库存不足、重复评价、未登录访问、空购物车结算等
